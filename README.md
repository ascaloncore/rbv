# RBV (Reading BookValue)
〜 知識の「不良在庫」をゼロにし、自己投資のROIを最大化する学習資産管理アプリ 〜

## サービス概要
RBV (Reading BookValue) は、購入したものの未消化となっている学習資産(いわゆる、積読や未読本)を、企業の「棚卸資産（在庫）」に見立てて管理するアプリです。
一般的な学習記録（時間管理）ではなく、「購入金額（コスト）」と「在庫状態(適正在庫・滞留在庫・不良在庫)」を可視化し、**埋没費用（損失）をユーザーに対して突きつける**ことで学習動機を生み出します。
「知識の不良在庫」をゼロにし、自己投資のROI（投資対効果）を最大化することを目的としています。

## このサービスへの思い・作りたい理由
**学習資産（財政状態）の「見える化」を通じたROIの最大化**

私は20年間、経理財務プロフェッショナルとして企業のディスクロージャー（決算開示）に関わる中、特にグループの財政状態の重要性を肌身で感じてきました。財政状態は、「現金としての回収可能額」を表現したものであり、企業は調達した負債（借入金）と資本（新株発行）により、投資リターンを最大化させ、企業活動の通信簿として財政状態及び経営成績として残した上、その結果には厳しい説明責任が求められます。
企業において「棚卸資産」は、KPI（在庫回転率や不良在庫率）を通じて厳しく評価されます。これを私個人の学習資産に置き換えた場合、私の学習資産はまさに**「経営破綻」状態**でした。

「いつかやる」と買い込んだ技術書、セールの勢いで買った動画講座……これらは「巨額の不良在庫」となり、キャッシュフローを悪化させていました。
**「自分という会社の経営なら、これは即刻倒産だ」**
この強烈な危機感が、本アプリの出発点です。

既存の学習アプリは「頑張った記録」をつけるものばかりですが、本アプリは**「無駄になった損失」を直視**させます。多忙な現代人が精神論ではなく、企業会計的・経営的なアプローチでコストを「血肉（資産）」に変えるための管理ツールです。

## ターゲット
**20代〜40代の読書習慣のある学生・エンジニア・ビジネスパーソン**

自己成長意欲が高く、書籍代や自己投資に月1万円以上を投資する経済力がある反面、多忙により「購入したことで満足する」罠に陥りがちな層。
彼らに響くのは、「頑張りましょう」という励ましではなく、**「現在、あなたは2,560,000円をドブに捨てています」という合理的かつ経済的なファクト**です。

## サービスの利用イメージ
本アプリは、ユーザーの学習状況を**B/S（貸借対照表）**のように管理し、「損失回避」「評価減からの脱却」をゴールに行動変容を促します。

1.  **衝撃 (Reality Check)**
    * ログイン直後のダッシュボードに、未了在庫の総額が**「在庫金額総額（リスク資産）」**として大きく表示します。ユーザーは自身の浪費状況を突きつけられます。
2.  **仕入 (Asset Entry)**
    * 在庫登録(購入書籍)を登録します。あえて手入力を基本とすることで、出費の痛みを自覚させます。
3.  **資産化 (Transformation)**
    * 読書を開始し、「未読」～「読書中」などステータスを更新します。読書完了により「知識資産」へステージアップし、初めて、その投資が「知識資産」へ計上され、ダッシュボードの「在庫金額総額」が減少します。
4.  **決算 (Closing)**
    * **「減損処理」機能**を利用し、読書見込みのない学習資産（在庫）を一括で評価減します。帳簿価額を1円（備忘価額）に切り下げ、「減損損失」として計上することで、過去の失敗と決別し、心理的な区切りをつけます。
5.  **価値 (Value)**
    * ユーザーは「不良在庫による損失（減損損失）を回避したい」という動機で読書を進め、自己のROIの最大化に向けて強い意志をもって日々を過ごし、結果として知識という資産を獲得します。

## ユーザーの獲得について
### 想定ユーザーへのアプローチ
* **エンジニアコミュニティ（RUNTEQ等）への展開**
「在庫一掃キャンペーン」と銘打ち、年末年始や年度末などの節目に、自身の積読状況を棚卸しするイベントとして利用を促します。
* **SNS（X/Twitter）でのバイラル**
「私の積読損失額は〇〇円でした #RBV」という、自虐的かつインパクトのあるシェア機能を実装し、エンジニア界隈での話題化を狙います。「学習時間」のシェア（ポジティブ）に対するアンチテーゼとして、「損失額」のシェア（ネガティブ・ユニーク）を定着させます。

## サービスの差別化ポイント・推しポイント
### 競合アプリ（Studyplus等）との差別化

| 比較軸 | 既存アプリ（Studyplus等） | 本アプリ (RBV) |
| --- | --- | --- |
| **管理対象** | 学習時間・継続日数 (Process) | 購入金額・在庫状態 |
| **視点** | フロー（どれだけ時間を流したか） | ストック（どれだけ資産が積み上がったか） |
| **動機付け** | ポジティブ（積み上げ・いいね） | ネガティブ（損失回避・評価減脱却） |

### 推しポイント：徹底した「経営視点メタファー」の世界観
* 在庫のステータス管理を「未読」「読書中」「停滞中」「中断中」「減損損失」「読書完了」など経営及び会計的視点による用語で定義。
* 事務的になりがちな学習タスク消化を、「ROI最大化」というミッションに変換し、ユーザーを「経営者視点」に引き上げます。
* 「引き算の美学」として、SNS機能やAPI連携をあえて排除し、孤独に数字と向き合う硬派なツールとしての立ち位置を確立します。

## 機能候補
### MVPリリース時に具備する機能郡
### ユーザー管理 (User Management)
- [x] **ユーザー登録機能**: 新規アカウントを作成できます。
- [x] **ログイン機能**: メールアドレスとパスワードで認証を行います。

### 在庫(書籍)管理 (Inventory Management)
- [x] **在庫登録機能 (仕入)**: 購入した書籍を「在庫」として登録します。
- [x] **在庫一覧・詳細閲覧機能**: 登録した書籍のステータスや「帳簿価額」を確認できます。
- [x] **在庫編集機能**: 書籍情報の修正が可能です（※減損処理後は一部制限あり）。
- [x] **在庫削除機能**: 誤って登録した在庫を削除できます。

### 読書メモ機能 (Reading Memos)
- [x] **メモ投稿機能**: 各書籍詳細ページにて、気づきや学びをメモとして残せます。
- [x] **メモ閲覧・編集・削除機能**: 過去のメモを振り返り、修正や削除が可能です。

### 資産管理・ダッシュボード (Asset Management & Dashboard)
- [x] **ダッシュボード表示**: TOPページにて**「在庫金額総額」「知識資産総額」「減損損失累計額」**をリアルタイムで可視化します。
- [x] **ステータス管理**: 書籍の状態（未読、読書中、読書完了など）を管理します。
- [x] **決算処理（減損損失処理）機能**: 読まなくなった書籍を一括で「減損処理」し、帳簿価額を1円に切り下げる機能です。
